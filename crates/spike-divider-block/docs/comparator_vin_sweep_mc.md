# T.11.E — Hybrid comparator vin-sweep × Monte Carlo (Apple Metal)

Standalone 9-transistor Baker-style comparator, **B = 4096 chips** (16 input offsets × 256 mismatch realizations) run through ONE `transient_pwl_batched` call. Per-chip `vp` (= V_CM + vin_offset) via the boundary closure; per-chip M1/M2 Vth (σ = 5 mV per side) via `mc_params`. Smaller graph than the SAR ADC means MLX can fully lower it (no fallback paths) and GPU dispatch dominates; intended as the clean correctness + perf demo of the T.11.D hybrid-batch architecture.

## Headline

- **4096 chips × 80 BE steps** in **18.7 s** (0.06 ms / step / chip)
- **Input-referred offset distribution (per-draw switching point at vout = V_DD/2)**:
- mean = **-2.12 mV**
- σ = **7.67 mV**
- Caught 256/256 crossings within ±50 mV sweep range; outside-window draws have offsets larger than the swept span (i.e. one-sided saturation)

## What this proves

- The **hybrid 2-axis batch** (vin sweep × MC draws) flows through one `transient_pwl_batched` call — boundary-per-chip and `mc_params`-per-chip both work end-to-end on a circuit MLX can fully lower.
- The σ of the per-draw switching point = **input-referred offset σ** of the comparator under random mismatch. With M1, M2 each at σ_Vth = 5 mV per side, the differential input pair sees σ_ΔVth ≈ 7 mV referred to vp - vm, which is what we measure here directly.
- `RLX_BATCHED_DEVICE=mlx` dispatches the entire residual + jacobian to the Apple Metal/MLX backend — small graph, fully lowered, no per-op CPU fallback. Compare with the SAR ADC bin (T.11.D) where the larger residual graph triggers partial fallback paths.
